NOMBRE
       raidtab  -  fichero  de configuración para dispositivos md
       (RAID)

DESCRIPCIÓN
       /etc/raidtab es el fichero de  configuración  por  defecto
       para  las  herramientas  RAID (raidadd y compañía). Define
       cómo se configuran los dispositivos RAID del sistema.


FORMATO
       /etc/raidtab tiene varias secciones, una para cada dispos­
       itivo  md  que  se configure. Cada sección comienza con la
       palabra reservada raiddev.  El orden de los  elementos  en
       el  fichero es importante. Una entrada raiddev pueden usar
       entradas raiddev definidas previamente  (lo  que  permite,
       por ejemplo, un RAID-10); además, el código de análisis no
       es excesivamente brillante, por lo que asegúrese de seguir
       el orden que aparece en esta página de manual para obtener
       los mejores resultados.

       Aquí tiene un ejemplo de fichero de configuración de md:

       # Ejemplo de fichero de configuración de dispositivos RAID
       raiddev /dev/md0
           raid-level                0
           nr-raid-disks             2
           nr-spare-disks            0

           device                    /dev/hda1
           raid-disk                 0
           device                    /dev/hdb1
           raid-disk                 1

       raiddev /dev/md1
           raid-level                5
           nr-raid-disks             3
           nr-spare-disks            0

           device                    /dev/sda1
           raid-disk                 0
           device                    /dev/sdb1
           raid-disk                 1
           device                    /dev/md0
           raid-disk                 2

       A continuación tiene más información sobre las  directivas
       que  aparecen  en  los ficheros de configuración RAID; las
       opciones se  muestran  aquí  en  el  mismo  orden  en  que
       deberían aparecer en el fichero de configuración real.


       raiddev dispositivo
              Esto introduce la información de configuración para
              el dispositivo indicado.


       nr-raid-disks número
              Número de dispositivos RAID en  el  array;  debería
              haber  número  entradas  raid-disk  después  en  el
              fichero.


       nr-spare-disks número

              Número de dispositivos de respuesto  en  el  array;
              debería haber número entradas spare-disk después en
              el fichero. Los discos de respuesto sólo se  pueden
              usar  con  RAID4  y RAID5 y permiten al núcleo con­
              truir automáticamente nuevos discos RAID cuando  se
              necesiten.  Dése  cuenta  que esta posibilidad está
              deshabilitada actualmente en el núcleo.


       parity-algorithm algoritmo
              El algoritmo de paridad a usar con RAID5. Debe  ser
              uno de los siguientes: left-asymmetric, right-asym­
              metric, left-symmetric o right-symmetric.


       chunk-size tamaño
              Establece el tamaño del stripe a tamaño bytes.


       device dispositivo
              Añade el dispositivo dispositivo a la lista de dis­
              positivos  que  comprenden  el  sistema  RAID. Dése
              cuenta que esta directiva debe  estar  seguida  por
              otra raid-disk, spare-disk o parity-disk.


       raid-disk índice
              El  device definido más recientemente se inserta en
              la posición índice del array RAID.


       spare-disk índice
              El device definido más recientemente se inserta  en
              la posición índice del array de discos de repuesto.


       parity-disk índice
              El device definido más recientemente  se  mueve  al
              final  del array RAID, lo que le obliga a ser usado
              para paridad.


NOTAS

       raidtools,  que originalmente fueron escritos pr Marc Zyn­
